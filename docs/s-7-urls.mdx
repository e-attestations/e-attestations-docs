For each environment, a specific authentication server will provide you with the JWT you need.

Environment | URL
------------|---------------------
Non PRODUCTION|`https://auth.test-e-attestations.com`
PRODUCTION|`https://auth.e-attestations.com`


> Get a JWT in TEST environement  with `curl`:

```sh
curl --request POST 'https://auth.test-e-attestations.com/auth/realms/eat_realm/protocol/openid-connect/token' \
--data-urlencode 'grant_type=password' \
--data-urlencode 'client_id=ea-api-edge' \
--data-urlencode 'username=<your user>' \
--data-urlencode 'password=<your password>' \
--data-urlencode 'Content-Type=application/json'
```
