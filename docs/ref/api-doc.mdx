## get__account_{accountId}_thirdparties

> Code samples

```http

Host: rs.e-attestations.com
Accept: application/json

```

`GET /account/{accountId}/thirdparties`

*Thirdparty list*

## Fetch thirdparties

On its simplest form, fetch thirparties with Pagination

Get thirdparties :
`/api/v1/account/{{account_id}}/thirdparties`

Using pagination :
`/api/v1/account/{{account_id}}/thirdparties?page=0&pageSize=1`

Various request parameters can be used for searching any particular thirdparty by : 

### For non French :

  - SIREN id : `/api/v1/account/{{account_id}}/thirdparties?siren=503829368`

  - SIRET id : `/api/v1/account/{{account_id}}/thirdparties?siret=43258335900109`

### For non French :

  - TVA id : `/api/v1/account/{{account_id}}/thirdparties?tva=B26492108`

<h3 id="get__account_{accountid}_thirdparties-parameters">Parameters</h3>

|monitored|query|boolean|false|Allows filtering on monitored third parties (monitored = true) or non-monitored third parties (monitored = false)|
|companyIdType|query|string|false|Third-party company identifier type (CIT001, CIT002 etc.).|
|name|query|string|false|Thirdpary name|
|siren|query|string|false|Thirdparty SIREN|
|siret|query|string|false|Thirdparty SIRET|

|page|query|integer|false|Paging parameter indicating the page number.|
|pageSize|query|integer|false|Pagination parameter indicating the desired page size.|

`POST /account/{accountId}/dossiers`

*Create/add a 'dossier'*

Create a 'dossier' for a thirdparty.

Launch e-Attestations internal processes for continuus monitoring for thirdparty compliancy.

Create a dossier for a new thirdparty (not already monitored) starts the invitation workflow (by email). Thus, all information are important during this phase.

This 'dossier' allows the client to monitor one of his thirdparty for one or a number of required documents.

Available documents depends on client subscription, configuration. Always ask you account manager for any questions. Documents are represented through 'document code' that you have to recognize in order to ask for the good requirements and reach evidences for this 'dossier'.

It's a veru powerful mecanism, giving a lot of flexibility and adaptability for the customer/system using APIs.

If the Thirdparty does not exist, it is created in the eAttestations platform.

'Dossier' creation is an **asynchronous process**. This asynchronism is induced by the process (itself of an asynchronous nature) of qualification of Thirdparty information by our internal data sources.

This endpoint launches the import process and returns a process identifier (processId).

The processing can last few minutes. In order to follow the progress of the import process and to know its result, a client can check the specific processid and know if the asynchronous creation process is complete and successful.
